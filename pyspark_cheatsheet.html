Quick Reference Guide

Reading data
spark.read.format("delta").load(path)Read a Delta table from a path

spark.read.csv(path, header=True, inferSchema=True)CSV - inferSchema is slow on large files; define schema manually in prod
spark.read.json(path)JSON - for multiline: .option("multiLine", True)
spark.read.parquet(path)Parquet - schema is embedded in the file, no inference needed
Writing data
.write.format("delta").mode("overwrite").save(path)Overwrite a Delta path
.write.mode("append").saveAsTable("name")Append to a registered table
.write.partitionBy("date").format("delta").save(path)Partition on write - good for time-based query pruning
Write modes

Join syntax lazy
# Standard join
df1.join(df2, on="station_id", how="inner")

# Multi-column key
df1.join(df2, on=["station_id", "date"], how="left")

# Expression key - avoids duplicate column names in output
df1.join(df2, df1.id == df2.station_id, "inner")
Join types
innerOnly matching rows from both sides
left / left_outerAll left rows; nulls where no right match
right / right_outerAll right rows; nulls where no left match
full / full_outerAll rows from both sides
left_semiLeft rows that have a match - like a filtered inner; no right columns in output
left_antiLeft rows that have no match - set difference
crossCartesian product - use with extreme caution
Joins on large tables cause shuffles. If one table is small (<200MB), Spark broadcasts it automatically. Force it with broadcast(df_small) imported from pyspark.sql.functions.
Window function setup
from pyspark.sql.window import Window
from pyspark.sql.functions import row_number, rank, dense_rank, lag, lead, avg

w = (Window
    .partitionBy("station_id")
    .orderBy("observation_ts")
)
Common window functions
row_number().over(w)Sequential number within partition - no ties
rank().over(w)Rank with gaps on ties (1, 1, 3…)
dense_rank().over(w)Rank without gaps on ties (1, 1, 2…)
lag("col", 1).over(w)Previous row's value
lead("col", 1).over(w)Next row's value
avg("temp").over(w)Running average over the partition + ordering
Silver-layer dedup pattern - keep latest per key
w = Window.partitionBy("station_id", "observation_ts").orderBy(col("ingested_at").desc())

(df
    .withColumn("rn", row_number().over(w))
    .filter(col("rn") == 1)
    .drop("rn")
)
Partition on the natural key, order by ingestion time descending, keep row 1. Lighter than MERGE when the source never reissues past records.

Structured streaming + foreachBatch
# Read a Delta table as a stream
df = spark.readStream.format("delta").table("bronze.weather")

# Run arbitrary batch logic per micro-batch
def upsert_to_silver(batch_df, batch_id):
    batch_df.createOrReplaceTempView("updates")
    batch_df._jdf.sparkSession().sql("""
        MERGE INTO silver.weather t
        USING updates s
        ON t.station_id = s.station_id AND t.observation_ts = s.observation_ts
        WHEN MATCHED THEN UPDATE SET *
        WHEN NOT MATCHED THEN INSERT *
    """)

df.writeStream
    .foreachBatch(upsert_to_silver)
    .option("checkpointLocation", checkpoint_path)
    .trigger(availableNow=True)
    .start()
Trigger modes
trigger(availableNow=True)Process all backlog then stop - serverless compatible
trigger(once=True)Deprecated - use availableNow instead
trigger(processingTime="30 seconds")Continuous micro-batch - not supported on serverless
Checkpoints are cumulative. File counts reflect all files ever seen by the stream, not current volume contents. Resetting requires deleting the checkpoint directory or pointing to a new path.
Partitioning & caching
df.repartition(8)Shuffle into N partitions - use before a wide join on an unpartitioned table
df.repartition(col("date"))Partition by column value
df.coalesce(1)Reduce partitions without a shuffle - only merges, never splits
df.cache()Persist in memory for repeated use - call .unpersist() when done
df.persist(StorageLevel.DISK_ONLY)Persist to disk when DataFrame is too large for memory
Broadcast join avoids shuffle
from pyspark.sql.functions import broadcast

df_large.join(broadcast(df_small), on="station_id")
# Sends the small DF to every executor - eliminates shuffle entirely
# Rule of thumb: use when one side is <200MB
Spark Performance Concepts
ShuffleData redistribution across partitions - triggered by joins, groupBy, repartition. The most expensive Spark operation.
Catalyst optimizerRewrites your query plan for efficiency before execution. Primary reason to prefer the DataFrame API over the RDD API.
PhotonDatabricks native execution engine - replaces JVM path for SQL and DataFrame ops. Same API, faster execution. Databricks-only, not open-source.
Predicate pushdownCatalyst pushes .filter() down to the data source so fewer rows are scanned. Automatic with Delta and Parquet.
AQEAdaptive Query Execution - Spark 3+ re-optimizes the plan at runtime based on actual partition sizes observed during execution.
What RDDs are and when to use them
RDDResilient Distributed Dataset - the low-level abstraction under DataFrames. Untyped, no schema, no Catalyst optimization. Use only when DataFrames can't express the logic.
DataFrame vs RDDDataFrames go through Catalyst and Tungsten - faster in almost every case. RDDs are useful for unstructured data, custom partitioning logic, or operations with no DataFrame equivalent.
scSparkContext - the entry point for RDD operations. In Databricks: sc = spark.sparkContext
Creating RDDs
sc.parallelize([1, 2, 3, 4])Create an RDD from a local Python collection
sc.parallelize([1, 2, 3], numSlices=4)Specify number of partitions explicitly
sc.textFile("s3://bucket/file.txt")Read a text file - one string per line
df.rddConvert a DataFrame to an RDD of Row objects
spark.createDataFrame(rdd, schema)Convert an RDD back to a DataFrame
Transformations lazy
rdd.map(lambda x: x * 2)Apply a function to every element - returns one output per input
rdd.flatMap(lambda x: x.split(" "))Like map but flattens the result - one input can produce many outputs
rdd.filter(lambda x: x > 0)Keep elements that match the predicate
rdd.distinct()Remove duplicates - triggers a shuffle
rdd.mapPartitions(func)Apply a function to each partition as an iterator - more efficient than map for setup-heavy ops
rdd.sortBy(lambda x: x[1], ascending=False)Sort by a key function
rdd.union(rdd2)Combine two RDDs - no deduplication
rdd.repartition(8)Shuffle into N partitions
rdd.coalesce(2)Reduce partitions without a full shuffle
Key-value (PairRDD) transformations lazy
rdd.map(lambda x: (x[0], x[1]))Create a PairRDD - any RDD of (key, value) tuples
rdd.reduceByKey(lambda a, b: a + b)Aggregate values per key - more efficient than groupByKey because it combines locally before shuffling
rdd.groupByKey()Group all values per key into an iterable - avoid on large data, use reduceByKey instead
rdd.mapValues(lambda v: v * 2)Apply a function to values only, preserving keys
rdd.sortByKey(ascending=True)Sort by key
rdd.join(rdd2)Inner join on key - results in (key, (v1, v2))
rdd.leftOuterJoin(rdd2)Left join - missing right values are None
rdd.keys() / rdd.values()Extract keys or values as a new RDD
rdd.countByKey()Count occurrences per key - returns a Python dict action
Actions action
rdd.collect()Return all elements as a Python list - dangerous on large RDDs
rdd.count()Number of elements
rdd.first()Return the first element
rdd.take(5)Return first N elements as a list
rdd.top(5)Return top N elements (sorted descending)
rdd.reduce(lambda a, b: a + b)Aggregate all elements using a commutative function
rdd.fold(0, lambda a, b: a + b)Like reduce but with a zero value for each partition
rdd.foreach(print)Apply a function to each element with no return value
rdd.saveAsTextFile(path)Write each element as a line to a text file
Persistence
rdd.cache()Persist in memory (MEMORY_ONLY) - shorthand for persist()
rdd.persist(StorageLevel.MEMORY_AND_DISK)Spill to disk if memory runs out
rdd.unpersist()Release cached data
rdd.getNumPartitions()Check how many partitions the RDD has
RDDs lack Catalyst optimization, are lower-level than DataFrames, and that reduceByKey is preferred over groupByKey because it aggregates locally before shuffling.
